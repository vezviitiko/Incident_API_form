# Incident_API_form
A python program for creating incidents on a local host with a specific form

**Задача**
Сделать маленький API-сервис для учёта инцидентов.
**Контекст**: операторы и системы присылают сообщения о проблемах в TG (самокат не в сети, точка не отвечает, отчёт не выгрузился). Не хотим **терять это в чатах.**

**Технологии:**
- Python
- FastAPI
- SQLAlchemy
- Pydantic

**Функциональность:**

Инцидент содержит:
    id - уникальный идентификатор
    description - описание проблемы
    status - статус (new, in_progress, resolved, closed)
    source - источник (operator, monitoring, partner)
    created_at - время создания

Требуется:

  1. **Создать инцидент**
    
  2. **Получить список инцидентов (с фильтром по статусу)**
     
  3. **Обновить статус инцидента по id**
     
      Если не найден — вернуть 404.

    Добавлено от себя
        4. Удаление конкретного id
        5. Удаление всех данных инцидентов

**Реализация и тесторивание**

1. Установите зависимости:
pip install fastapi uvicorn sqlalchemy pydantic

2. Запустите сервер:
python main.py

3. Выполните запрос в postman (его конструкт в отдельном файле)

4. Откройте в браузер:
http://127.0.0.1:8000/incidents/

5. Запустите программа тестирование функционала:
python test.py

6. Сравните результаты
Сравните полеченные результаты из предыдущих 5 действий с логом тестирования из файла test.log
